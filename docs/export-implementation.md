# Export implementation (Phase 7)

## Renderers

- **PDF** — `pdfkit` (`lib/pdfRenderer.ts`). A4, margins from
  `@workspace/export-contract` constants. Watermark + footers applied per
  page in a final pass via `bufferedPageRange`.
- **Word** — `docx` (`lib/wordRenderer.ts`). Same A4 geometry expressed in
  twips (`mmToTwip`). Watermark lives in the page header so it repeats on
  every page; running footer carries page numbers.

Both renderers are **pure**: they accept an `AnnualReportExportData` and
return a `Buffer`. They never read the database, never log audit events,
and never make decisions about whether to watermark — those are made by the
data builder and route layer.

## Routes

All under `/api`:

| Verb | Path | Purpose |
| --- | --- | --- |
| GET  | `/reports/:reportId/export/data`      | Snapshot for preview |
| GET  | `/reports/:reportId/export/readiness` | Block / warn / ok verdict |
| POST | `/reports/:reportId/export/cover`     | Update cover sheet |
| POST | `/reports/:reportId/export/pdf`       | Generate PDF |
| POST | `/reports/:reportId/export/word`      | Generate Word |
| GET  | `/reports/:reportId/export/history`   | Past exports |

`/projects/:projectId/export/{data,readiness,history}` are mirror endpoints
that resolve the matching report via
`projectReportLink.resolveReportForProject` and delegate.

## Storage & download

Generated bytes are uploaded to Supabase Storage (`exports` bucket for paid
projects, `demo-assets` for demo). The existing `/api/exports/:id/download`
endpoint mints a 1-hour signed URL via
`supabase.storage.from(bucket).createSignedUrl(path, 3600)` and returns it
to the client.

When Supabase isn't configured (dev / preview), the bytes are also placed
in an in-memory cache keyed by the export id; the download endpoint streams
them inline. Old exports for which neither source resolves return a 410 so
the user knows to regenerate.

## Permission gates

- **Read** (`data`, `readiness`, `history`) — `canViewProject`.
- **Write** (`cover`, generate) — `canExportProject` (owner role).
- **Final (non-watermarked) generate** — additionally requires
  `hasPaidProjectEntitlement`. Demo projects always watermark.

## Audit events

Added in `helpers/auditLog.ts`:

- `export.preview_viewed`
- `export.readiness_checked`
- `export.cover_updated`
- `export.failed`
- (existing) `export.generated`, `export.downloaded`

## ID bridging

The codebase has two id worlds: `reportId` (authoring data) and `projectId`
(entitlement, demo, files, audit, exports). `helpers/projectReportLink.ts`
resolves between them by matching `(companyId, fiscalYearStart,
fiscalYearEnd)`. Reports without a paired project row degrade gracefully
(streaming download, empty history, no role gate).
