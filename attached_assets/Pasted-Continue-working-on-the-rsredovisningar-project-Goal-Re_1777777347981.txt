Continue working on the “Årsredovisningar” project.

Goal:
Redesign the public Demo experience on the website so it feels like a lightweight, polished product preview instead of a separate mini-version of the product.

The current Demo functionality should NOT be deleted. Move the current interactive demo functionality into Admin so it can later be used as an internal/customer demo environment.

Important product direction:
- The public website demo should feel like part of the homepage/marketing site.
- It should be easy to understand for accountants and small business owners.
- It should not feel like a fake workspace or a separate product mode.
- The purpose is to quickly show what the system can do and encourage users to register/pay/start a real annual report project.
- Free/demo users should still not get access to real editing, importing, validation, collaboration, or clean export.

Build on the current app structure.
Do not replace the whole app.
Reuse existing routes, components, design tokens, layout system and styling where possible.
Do not change the real annual-report workflow.
Do not build final real PDF export logic here.
Do not connect Stripe/payment logic in this phase unless already required by existing architecture.

## 2. Required demo slides

Create demo slides for the following areas:

### Slide 1: Dataimport – SIE
Show how the user can import accounting data.

Content direction:
- Show SIE import as the main example.
- Mention Excel/CSV as supported/coming depending on existing product wording.
- Show a clean visual of uploaded accounting data being interpreted.
- Include plain-language text:
  “Importera bokföringsdata från SIE och låt systemet föreslå struktur, mappning och årsredovisningsunderlag.”

Suggested visual elements:
- SIE file card
- import status
- account mapping preview
- “Konto 3001 → Nettoomsättning”
- “Konto 1930 → Bank”
- simple progress/check states

### Slide 2: Balansräkning & resultaträkning
Show BR/RR together in a polished way.

Content direction:
- Balance sheet and income statement can overlap visually in a clean stacked-card layout.
- The purpose is to show that imported accounting data becomes financial statements.
- Include the “Not” column visibly.
- Show that note references are connected to rows.

Suggested visual elements:
- two overlapping document/table cards:
  - Resultaträkning
  - Balansräkning
- visible columns:
  - Belopp
  - Föregående år
  - Not
- examples:
  - Nettoomsättning — Not 1
  - Materiella anläggningstillgångar — Not 3
  - Eget kapital — Not 5

Text:
“Se hur bokföringen omvandlas till balansräkning och resultaträkning med tydliga notkopplingar.”

### Slide 3: Kassaflödesanalys
Show cash flow statement support.

Content direction:
- Explain that the system can assess whether a cash flow statement is needed and help build it from underlying data.
- The user must later be able to review and adjust.
- Do not imply the system guarantees correctness.

Suggested visual elements:
- assessment card:
  - “Kassaflödesanalys: Obligatorisk / Frivillig / Behöver bedömas”
- cash flow sections:
  - Löpande verksamhet
  - Investeringsverksamhet
  - Finansieringsverksamhet
- adjustment hint:
  - “Manuell justering möjlig”
  - “Källa: importerade konton + omklassificering”

Text:
“Systemet hjälper till att bedöma, skapa och granska kassaflödesanalysen – med möjlighet till manuell justering.”

### Slide 4: Noter
Show notes and note numbering.

Content direction:
- This slide is important.
- Demonstrate that notes are connected to BR/RR rows.
- Show that note numbering is automatic and validated.
- Show that the user can review, include/exclude where appropriate, and edit content.

Suggested visual elements:
- note list:
  - Not 1 Nettoomsättning
  - Not 2 Anställda och personalkostnader
  - Not 3 Materiella anläggningstillgångar
  - Not 4 Långfristiga skulder
- linked financial statement row:
  - “Materiella anläggningstillgångar → Not 3”
- validation badge:
  - “Notreferenser matchar”
  - “Inga brutna hänvisningar”

Text:
“Noter kopplas till rätt rader i rapporten och kontrolleras så att numrering och hänvisningar stämmer.”

### Optional Slide 5: Färdig årsredovisning
If it fits naturally, add a final slide showing the finished annual report preview.

Suggested visual elements:
- cover page
- table of contents / sections
- export preview
- watermark “DEMO”

Text:
“Förhandsgranska en färdig svensk årsredovisning och ladda ner ett demoexempel.”

## 5. UX and content requirements

The new public demo should be simple and marketing-friendly.

Design requirements:
- Use cards, soft shadows, clear spacing, rounded corners.
- Avoid dense tables unless simplified.
- Avoid too many buttons.
- Use short text blocks.
- Make each slide instantly understandable.
- Use Swedish as the main language.
- Support existing language system if the website already supports Swedish/English.
- Annual report examples should be Swedish.
- Avoid fake overpromises such as “100% korrekt” or “garanterat godkänd”.
- Use wording such as:
  - “hjälper dig”
  - “föreslår”
  - “granska och justera”
  - “demoexempel”
  - “ej giltig årsredovisning”

The Demo must clearly communicate:
- import accounting data
- review mapping/structure
- generate BR/RR
- handle cash flow assessment/statement
- connect notes correctly
- preview/download demo annual report
