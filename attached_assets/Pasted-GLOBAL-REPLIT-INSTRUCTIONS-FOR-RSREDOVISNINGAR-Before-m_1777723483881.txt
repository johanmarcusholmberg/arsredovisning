You are working on the project “Årsredovisningar”.

Context:
This is a web application for creating Swedish annual reports with AI-supported guidance. The product should support both accountants/consultants and small business owners, but all users follow the same annual-report workflow. Do not create separate “simple” and “expert” product modes. Instead, use one unified workflow with adaptive guidance and progressive disclosure.

Important product principles:
- One unified annual-report workflow for all users.
- No reduced compliance mode based on user type.
- K3 is the primary MVP framework.
- K2 should also be supported.
- Free users should not be able to create real reports, import real accounting data, edit report content, collaborate on real reports, run real validation, or export clean documents.
- Free users should only access a demo workspace with dummy values and a sample Swedish K3 annual report.
- Payment is required before creating or working on a real annual-report project.
- Payment unlocks one real annual-report project for one company and one financial year.
- Website/application UI should support Swedish and English.
- Annual report output itself should be Swedish only at launch.
- Use temporary branding, logo and imagery placeholders.
- Design direction: “Financial Stripe + Swedish minimalism”.
- Product positioning: “Årsredovisningar – snabbt, korrekt och med AI-stöd.”

Important annual-report output principle:
The final report must follow Swedish annual-report conventions. The app should prepare for note numbering and note references throughout the report, even if the full note engine is implemented later. The demo workspace should show examples of note references in the income statement and balance sheet.

Your task in this phase:
Build the product shell, landing page, authentication-ready structure, demo workspace and payment-gated real project creation flow.

Do not build the full annual-report engine yet.
Do not build real SIE import yet.
Do not build full mapping, notes, validation, collaboration, or export yet.
Only create the foundation needed for those future phases.
