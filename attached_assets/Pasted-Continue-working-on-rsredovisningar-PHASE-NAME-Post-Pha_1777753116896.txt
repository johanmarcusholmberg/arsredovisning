GOAL:
Update the application’s handling of Swedish cash flow statements (“kassaflödesanalys”).

The system must:
1. Determine whether a cash flow statement is mandatory, optional, or unknown.
2. Explain the requirement to the user in plain Swedish.
3. Generate a cash flow statement from imported accounting data where possible.
4. Allow guided manual adjustments where source data is incomplete.
5. Validate that opening cash + cash flow for the year = closing cash.
6. Ensure cash and cash equivalents match the balance sheet.
7. Integrate the validated cash flow statement into the existing PDF/Word export and preview.
8. Block export when a mandatory cash flow statement is missing, invalid, or not reviewed.

LEGAL / ACCOUNTING LOGIC:
Implement the following rule model.

A cash flow statement is mandatory when:
- The company is classified as a “större företag”.
- The entity is a bostadsrättsförening, if this legal form is supported in the app.
- The project is a group annual report where cash flow statement is required.

For Aktiebolag and ordinary annual reports:
A company should be classified as “större företag” when it is listed on a regulated market OR when more than one of the following three thresholds are exceeded for each of the two most recent financial years:
- Average number of employees > 50
- Balance sheet total > 40,000,000 SEK
- Net revenue > 80,000,000 SEK

If the app does not have enough information to determine this automatically, show a “Needs review” state and ask the user to confirm missing values.

Important:
- Do not assume that K3 alone always means cash flow statement is mandatory.
- Do not assume that K2 alone always means cash flow statement is forbidden.
- The decision should be based primarily on company size, legal form, and report type.
- Smaller companies may choose to include a voluntary cash flow statement if the product supports it.
- If voluntary cash flow statement is enabled, clearly label it as voluntary in the workflow, but the exported report should still use normal Swedish annual-report terminology.

Use the existing import, SIE/CSV/Excel parsing, chart-of-accounts mapping, report mapping, and financial statement logic where available.

Create a cash-flow mapping layer that can infer:

VALIDATION RULES:
Extend the existing validation engine.

1. Requirement validation:
- If cash flow statement is mandatory but missing, block final validation and export.
- If cash flow requirement is unknown, block final validation and export until user confirms company size/legal requirement.
- If optional and disabled, allow export but show that no cash flow statement will be included.
- If optional and enabled, validate it with the same reconciliation rules as mandatory cash flow.

2. Reconciliation validation:
- Opening cash + total cash flow for year must equal closing cash.
- If difference ≠ 0, show blocking error.
- Apply the project’s existing rounding settings and tolerance.
- Show exact difference and affected lines.

3. Balance sheet consistency:
- Cash and cash equivalents in the cash flow statement must match the balance sheet cash/bank value.
- If the annual report uses rounded values, reconcile using the same rounding model as the existing export.
- Do not export a cash flow statement that contradicts the balance sheet.

4. Source completeness validation:
Warn if:
- Cash/bank accounts are missing or unmapped.
- Previous-year balance sheet data is missing.
- Fixed asset movements cannot be derived.
- Tax paid cannot be calculated reliably.
- Loans/equity/dividends are detected but not classified.

5. Manual adjustment validation:
- Require explanation for every manual adjustment.
- Store previous value, new value, user, timestamp, and reason.
- Show manual adjustments in the review panel.

6. Export validation:
- Existing PDF/Word export must refuse clean export if mandatory cash flow is missing or invalid.
- Existing demo/watermarked export must also show the correct state.
- Payment/export unlock must not bypass cash-flow validation.
