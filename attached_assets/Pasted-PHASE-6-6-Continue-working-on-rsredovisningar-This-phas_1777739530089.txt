## 1. Formal annual report structure

Use this document structure as the source of truth:

1. Optional cover sheet / generated cover page
2. Förvaltningsberättelse
3. Resultaträkning
4. Balansräkning
5. Kassaflödesanalys, only if required or explicitly enabled
6. Noter
7. Underskrifter
8. Optional export-package appendices, separate from the formal annual report:
   - Validation summary
   - Audit/change summary

The formal annual report should not include validation or audit appendices unless the user explicitly selects them as separate export-package appendices.

The formal annual report should not include internal audit/debug details by default.

Internal traceability should remain available in the application, but the formal annual report should only show the final approved annual-report presentation.

## 12. Export/compliance wording rule

The application may help users prepare, review and export a Swedish annual report.

The application must not claim:

- legally approved
- guaranteed compliant
- 100% correct
- error-free
- approved by Bolagsverket

Use safe wording:

- “No blocking validation issues were found. Please review before filing.”
- “Ready for review”
- “Ready for export — no blocking issues found”
- “Export is blocked because required issues remain.”

The exported document should follow Swedish annual-report structure and common layout conventions, but the product remains an assistant and review tool.
