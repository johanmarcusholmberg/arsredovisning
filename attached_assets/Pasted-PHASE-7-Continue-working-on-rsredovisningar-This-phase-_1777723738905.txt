## 2. Final export readiness check

Before export, run final validation.

Export must be blocked if:

- Blocking validation issues exist
- Required notes are incomplete
- Required notes are not confirmed
- Note numbering is missing, duplicated or invalid
- Financial statement line references a missing/broken note
- Notes marked not applicable are still referenced
- Hidden or excluded notes are still referenced
- Note totals do not reconcile to linked income statement or balance sheet lines
- Approved reclassifications/netting create inconsistent statement or note totals
- Rejected reclassification suggestions affect the export data
- Undone reclassifications affect the export data
- AI-generated text is unconfirmed
- Required company data is missing
- Required report sections are incomplete
- User does not have paid entitlement
- User lacks permission
- Export storage is unavailable
- Export data contract cannot be built safely

Use wording:

“Export is blocked because required issues remain.”

When ready:

“No blocking validation issues were found. Please review before filing.”

The readiness check must call or reuse:

- validation engine
- note reference consistency check from Phase 6.6
- reclassification/netting consistency check from Phase 6.6
- permission helpers
- entitlement helpers
- export data builder from Phase 6.6

Do not duplicate this logic inside PDF/Word generation.

## 4. Note numbering and references in export

Implement export logic so that:

- Note numbers are generated or refreshed before export using the central note-numbering logic.
- Note references in income statement match the notes section.
- Note references in balance sheet match the notes section.
- Cash flow references match where applicable.

- Hidden/excluded/not_applicable notes are not referenced.
- Manual notes are included only if valid and included_in_report is true.
- Manual notes receive valid automatic note numbers if included.

Critical:
Do not store note numbers as manually typed disconnected text in export.

Use note IDs and central numbering logic.

## 5. Approved reclassification/netting in export

The export must use final approved presentation amounts.

Export calculations must follow:

final presentation amount =
imported mapped amount
+ approved reclassifications into the row/node
- approved reclassifications out of the row/node

For approved netted rows:

net amount =
gross positive balances
+ gross negative balances

The export must:

- include approved reclassifications in final presentation amounts
- include approved netting in final presentation amounts
- exclude rejected suggestions
- exclude not_relevant suggestions
- exclude undone reclassifications
- preserve internal traceability in export metadata/audit history
- ensure note totals match linked statement lines
- ensure balance sheet still balances

The formal annual report should show the approved final presentation.

It should not show technical source-account/audit details by default.

Validation/audit/change summaries may be included as optional export-package appendices, not as part of the formal annual report unless explicitly selected.
